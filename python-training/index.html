Day 1 | GovernanceConcept

Governance guardrails for the training labs

Allowed in our training case
  • Pseudonymised customer and account IDs.
  • SAR values, channel codes, product families, regions.
  • Only the fields required to answer the business question.
Intentionally absent
  • Names, national IDs, full card numbers, full addresses.
  • Any live or customer-identifiable data.
  • Uncontrolled exports and unnecessary columns.

We are reinforcing a simple habit: minimal-field thinking. Pull what you need, justify it, and document any exclusions.

Day 1 | FunctionsDemo

Functions as reusable transformations

def clean_amount(amount_text):
    if pd.isna(amount_text):
        return np.nan
    cleaned = str(amount_text).replace(",", "").strip()
    if cleaned == "":
        return np.nan
    return float(cleaned)
Function checklist
  • Clear name and single job.
  • Explicit input and output behaviour.
  • Safe handling of blanks and bad values.
  • Easy to test on small examples.

Optional extension: packaging notebook logic as functions

def build_branch_kpi(transactions_df):
    required = ["txn_id", "branch_id", "fee_sar", "amount_sar", "status"]
    missing = [c for c in required if c not in transactions_df.columns]
    if missing:
        raise ValueError(f"Missing columns: {missing}")
    return (
        transactions_df[transactions_df["status"] == "POSTED"]
          .groupby("branch_id")
          .agg(txn_count=("txn_id", "count"))
          .reset_index()
    )
